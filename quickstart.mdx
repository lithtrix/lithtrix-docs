title: Quickstart
description: Go from zero to searching in three API calls.
---

## Step 1 — Discover

```bash
curl https://lithtrix.ai/v1/capabilities
```

Returns endpoints, rate limits, and credibility scoring rules. No auth required. The JSON includes **`version`** (**`2.1.0`** after Arc 11 feedback) — use it to detect discovery schema generation for tooling.

## Step 2 — Register

```bash
curl -X POST https://lithtrix.ai/v1/register \
  -H "Content-Type: application/json" \
  -d '{"agent_name":"my-agent","owner_identifier":"you@example.com"}'
```

Optional: add `"referral_agent":"<referrer-uuid>"` to credit another agent — use the UUID they see as `referral_code` on [`GET /v1/me`](https://lithtrix.ai/v1/me) (free tier: +100 lifetime web-discovery calls per qualifying referral for the referrer).

Returns your `ltx_` API key. **Store it immediately** — it is shown only once.

`agent_name` must contain only letters, digits, hyphens, and underscores.

## Step 3 — Search

```bash
curl "https://lithtrix.ai/v1/search?q=Singapore+climate+policy+2025" \
  -H "Authorization: Bearer ltx_your_key"
```

Returns credibility-scored results and your remaining quota. Successful JSON includes **`_lithtrix.search_id`** (UUID) — use it with **`POST /v1/feedback`** (`ref_type`: `search_id`, `ref_id`: that UUID) to send optional helpful / unhelpful / wrong signal. See [Feedback](/api-reference/feedback) and [`GET /v1/feedback/stats`](https://lithtrix.ai/v1/feedback/stats).

## What's in the Response

Every search result includes a `credibility_score` (0.5–1.0):

| Score | Source type |
|-------|-------------|
| 1.0 | `.gov` domains |
| 0.9 | `.edu` domains |
| 0.8 | Major news (BBC, Reuters, AP) |
| 0.7 | `.org` domains |
| 0.5 | All other sources |
